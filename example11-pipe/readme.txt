-pipe                    Use pipes rather than intermediate files.
-pipe is an efficient compilation option in GCC. Its main purpose is to use pipes instead of temporary intermediate files to pass data between compilation stages.
For example, the assembly code generated by cc1 can be sent directly to as through a pipe, rather than first being written to a .s file and then read again.
This reduces disk I/O and can speed up compilation, especially when building large projects with many source files.
